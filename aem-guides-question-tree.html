Q

"What is DITA and do our authors need to learn XML?"

Technical

Recommended Response

DITA (Darwin Information Typing Architecture) is an open, industry-standard XML framework for structuring content into reusable, topic-based modules. But here's the key thing — your authors don't need to know XML or see angle brackets. AEM Guides provides a web-based editor that looks and feels like a modern word processor. Authors work in a visual interface with familiar formatting tools, while DITA structure is handled behind the scenes. Power users can switch to a full-tags view if they want, but subject matter experts and casual contributors never need to touch code.

Q

"How does content reuse actually work in AEM Guides?"

Technical

Recommended Response

Content reuse in AEM Guides operates at multiple levels of granularity. At the broadest level, you have map-based reuse — a DITA map is like a table of contents that pulls topics together into a publication, and the same topic can appear in multiple maps without duplication. Then there's content references (conrefs) for fragment-level reuse — reuse a single paragraph, warning note, or procedure step across dozens of documents. Key references let you define variables (product names, version numbers, URLs) that resolve at publish time, so you can produce variant documentation for different products or regions from the same source. And conditional tagging lets you mark content for specific audiences, platforms, or product SKUs and filter at publish time. A single update to a reused component propagates everywhere automatically.

Q

"How does AEM Guides handle version control and collaboration?"

Technical

Recommended Response

AEM Guides has enterprise-grade version control built in — every save creates a new version with full history, diff/merge capabilities, and the ability to compare any two versions side-by-side. You can create baselines (point-in-time snapshots) that lock a specific version of every topic in a map, which is essential for regulatory compliance and audit trails. For collaboration, there's a web-based review workflow where reviewers can comment inline, suggest changes, and approve content — all with role-based access control. Multiple authors can work on different topics simultaneously without stepping on each other's work because content is modular at the topic level.

Q

"How does translation workflow work?"

Technical

Recommended Response

AEM Guides integrates with both human and machine translation services through automated workflows. Because content is structured in DITA at the topic level, you only translate what's changed — not the entire document. The system tracks which topics have been modified since the last translation and sends only those for re-translation. Combined with content reuse (translated content that's reused doesn't need re-translation), organizations typically see 30–60% reduction in translation volume and cost. It supports all major translation management systems and connectors.

Q

"Can we maintain separate branches for different product versions?"

Technical

Recommended Response

Combined with conditional tagging (where you flag content for specific product versions or SKUs), you can manage extremely complex documentation matrices without duplicating your entire content library. This is one of the areas where AEM Guides really outperforms traditional CMS and wiki-based documentation tools.
